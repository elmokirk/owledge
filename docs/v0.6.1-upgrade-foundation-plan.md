**Status:** draft (pending implementation)

## Problem Statement (why this exists)

Found during the v0.6.0 post-publish review (2026-06-24). Six blind spots confirmed in code:

1. **No upgrade path.** `init-project` uses `copy_file_if_missing` (`tools/owledge.py:181-186`) which silently skips existing files. Re-running on an installed project returns `skipped_existing` and updates nothing. No `upgrade`/`migrate`/`sync` subcommand exists among the 12 registered (`tools/owledge.py:1751-1839`).
2. **No version stamp.** Repo-wide search for `KIT_VERSION`/`owledge_kit_version`/`installed_version` returned zero hits. `kit-manifest.json` (`build_project_folder_kit.py:297-302`) has only `{generated_at, source, files[]}`.
3. **Multi-source version drift.** `VERSION`=`0.6.0`, `templates/agent-memory/README.md:4`=`version: "0.4.0"`, `PROJECT_CONTEXT.template.md:4`=`version: "0.1.0"` — four independent version strings, none derived from a single source.
4. **Drift detection only covers maintainer tree.** `source_vs_target_audit` (`agent_memory_cli.py:5330-5364`) audits `templates/agent-memory/` inside the Owledge repo, not user projects. `doctor` (`agent_memory_cli.py:2128-2185`) only sees files under the project root — a broken global-layer reference is invisible.
5. **Dogfood can drift silently.** No gate compares `internal/agent-memory/templates/` against `templates/agent-memory/templates/`. Edits to product templates don't propagate to the dogfood tree; no gate fails.
6. **No conceptual audit loop.** The project has no skill that stress-tests its own concepts (distribution, lifecycle, coherence) the way this review did manually.

**Decision C:** ship E1-E6 + the skill as **v0.6.1** before resuming v0.6.0 feature work. The first schema change (FB-016 `quick_read`) MUST NOT escalate its stale-detection gate from `warning` to `error` (`docs/v0.6.0-implementation-plan.md:259`) until E3 ships — amended in Q7 below.

## Phase 3 — E3: `owledge upgrade --dry-run` / `--apply`

**Depends on E1 + E2. Sequential.**

### Hard rules (baked into code, not flags)

- **Updatable allow-list** = runtime union of `ROOT_FILE_MAP` (`tools/owledge.py:84-90`) + `HOST_TOOL_FILES` (`:92-97`) + `HOST_SKILL_DIRS` (`:99-104`) + `templates/agent-memory/**` + `skills/**`. Computed from the SAME constants `init_project` uses — no parallel inventory.
- **Never-touch list** (Q2, hardcoded, no flag bypasses): `PROJECT_CONTEXT.md`, `AGENTS.md`, `CLAUDE.md`, `USER_CONTEXT.md`, `agent-memory/{decisions,plans,sessions,evidence,handoffs}/**`, `global-memory/**`. Assert checked BEFORE any write.
- **Modes** (`--mode=safe|force-templates|manual`):
  - `safe` (default, Q3): only overwrite files where `sha256_installed == sha256_original` (pristine). Skip user-edited files with per-file warning. Never touch never-touch list.
  - `force-templates`: overwrite everything in updatable allow-list EXCEPT never-touch list. Requires interactive confirmation or `--yes`.
  - `manual`: emit per-file unified diffs + single `git apply`-able patch to stdout / `agent-memory/exports/upgrade-pending.patch`. No writes.

### Files

- `tools/owledge.py:~1804` — `upgrade_p = sub.add_parser("upgrade", parents=[project_parent])` with `--dry-run` (default True), `--apply`, `--mode`, `--yes`, `--format`.
- `tools/owledge.py` — `upgrade_project(root, dry_run, mode, yes)`:
  - Load `kit-manifest.json`. Absent → error "run `init-project` first". Graceful degradation (red-team #1).
  - Classify each file: `outdated` / `user_edited` / `current` (reuse E2 logic).
  - Stage new template content from running CLI's bundled templates.
  - Apply mode rules; `--dry-run` prints report; `--apply` writes.
  - File lock `agent-memory/.upgrade.lock` (PID + timestamp) to refuse concurrent runs (red-team #7).
  - On success, rewrite `kit-manifest.json` with new `kit_version` + refreshed hashes + `--author` field (red-team #3).
  - Additive-change awareness (red-team #6): if `CHANGELOG.md` declares the version change `breaking: no|additive`, `--dry-run` only reports informationally, doesn't alert.
- `tools/owledge.py:~1948` — dispatch branch `if args.command == "upgrade":`.
- `docs/upgrading.md` (minimal stub, Q9) — version stamping explainer + mode matrix + never-touch guarantee + recovery (`git checkout` manifest + re-run `--dry-run`).

### Acceptance

`upgrade --apply --mode=safe` preserves user-edited files (byte-identical) and updates pristine files. `--mode=force-templates --yes` leaves every never-touch file byte-identical. `--mode=manual` emits a `git apply --check`-valid patch. Second `--apply` run reports zero changes (idempotent).

### Tests (stress — most critical in the whole plan)

- `upgrade-safe-preserves-edits`: init at v0.6.0, edit `AGENTS.template.md` hash, bump to 0.6.1, `upgrade --apply --mode=safe` → edited file UNCHANGED, pristine files UPDATED, manifest rewritten, `doctor` green.
- `upgrade-force-templates-respects-never-touch`: same setup, `upgrade --apply --mode=force-templates --yes` → all 4 user-content files (`PROJECT_CONTEXT.md`/`AGENTS.md`/`CLAUDE.md`/`USER_CONTEXT.md`) byte-identical before/after (hash compare).
- `upgrade-manual-emits-patch`: `upgrade --dry-run --mode=manual` → `git apply --check` succeeds.
- `upgrade-idempotent`: run `--apply` twice → second run zero changes.
- `upgrade-manifest-deleted-graceful`: delete `kit-manifest.json`, `upgrade` → friendly error, no crash (red-team #1).
- `upgrade-concurrent-run-blocked`: hold `.upgrade.lock` with fake PID, second `--apply` → refuses.

### New gate

- `tools/owledge.py:~1402` — `add("upgrade-drift", lambda: core.upgrade_drift_check(root))` running `doctor --check-version` on a temp-init'd project at current version → must pass.

### E3 gate

Inter-step gate block + all 6 stress tests + `finalization-gates` green including `upgrade-drift`.

### The 8 audit dimensions (Decision B: 1-4 mechanical, 5-8 guided)

| # | Dimension | Type | Depth scales with `project_mode` |
| --- | --- | --- | --- |
| 1 | Lifecycle & upgrade | **Mechanical** — checks `init-project`/`upgrade`/`doctor`/uninstall paths exist & tested | poc: shallow → saas: +rollback +multi-env |
| 2 | Distribution integrity | **Mechanical** — verifies every install shape (PyPI/source/kit/plugin) consistent via `sdist-clean` + `release-trust` | poc: 1 shape → saas: +SBOM +provenance |
| 3 | Dogfood fidelity | **Mechanical** — `dogfood-sync-check` + skill-content hash gate | poc: spot → saas: +telemetry parity |
| 4 | Contract completeness | **Mechanical** — every CLI subcommand / schema field has a test + doc + gate | poc: list → saas: +every public API |
| 5 | Cross-layer integrity | **Guided** — produces checklist + evidence packet for agent/user to work through | poc: n/a → saas: +consent +audit log |
| 6 | Failure-mode coverage | **Guided** — enumerates breakages per dependency/path/consent; drills per audience | poc: list → saas: +chaos drill |
| 7 | Conceptual coherence | **Guided** — glossary + term-frequency + synonym audit | poc: glossary → saas: +cross-team term contract |
| 8 | Self-description accuracy | **Guided** — README/AGENTS/CHANGELOG claims vs actual behavior | poc: spot → saas: +traceability matrix |

Weights are tunable via `agent-memory/concept-audit-profile.json`; defaults sensible per `project_mode`.

### Delivery (adapts to `planning_mode`)

- `supervised` (default): findings inline, no files, no ideas/tickets.
- `approve-automatically`: writes `concept-audit-YYYY-MM-DD.md` to `agent-memory/decisions/`; promotes each `fail` finding to a draft idea card (`agent-memory/ideas/`, `status: captured`, edges to report). No canonical promotion — needs curator review.
- `full-access`: above + opens a feedback ticket via `docs/feedback-round-YYYY-MM.template.md` (FB-012) when `project_mode >= mvp`.

### Acceptance — the META test (most important)

**`skill-finds-its-own-gaps`**: run `concept-audit` against owledge itself at `project_mode: mvp` → the audit MUST surface ≥4 of the 6 findings from this session's analysis (no upgrade path, multi-source version drift, dogfood drift potential, undetected global-link, silent-skip init, no conceptual audit loop). If it doesn't, the skill is incomplete. This is the skill's acceptance gate — it must reproduce the gaps this plan exists to fix.

### Additional tests

- `skill-adapts-to-mode`: `poc` audit shorter (shallow); `saas` audit fails without provenance/SBOM/chaos drills.
- `skill-respects-planning-mode`: `supervised` → no files written; `approve-automatically` → draft idea cards with `status: captured`, no canonical promotion.

### Skill gate

`test-contracts` passes with new skill in `REQUIRED_DIRS`/`REQUIRED_FILES` + `skill-finds-its-own-gaps` + the two additional tests + `finalization-gates` green including `concept-audit-fresh`.
