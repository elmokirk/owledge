## Session-Continuity Convention

This plan uses per-phase checklists. Each phase has three checkboxes:

- `- [ ] implementation done`
- `- [ ] QA checks done`
- `- [ ] quick review done`

**Resume rule:** a new agent reads this plan, finds the first unchecked box,
and continues from there. Never restart a completed phase. If a session breaks
mid-phase, re-run that phase's QA gate before continuing; if it fails, uncheck
the box and redo the phase. Subagents check their own boxes before returning.

## Critical Correction to the Red-Team Review

**P0-1 (corrected).** `HOST_SKILL_DIRS` (`tools/owledge.py:112-117`) lists only
4 skills and does **NOT** include `skills/concept-blindspot-audit` — the

(`:241-280`) never walking `HOST_SKILL_DIRS` or `skills/**`, the skill is:

(c) not drift-detected by `doctor`, (d) not upgradable by `upgrade`. The fix
has two coupled parts: extend `HOST_SKILL_DIRS` to include the new skill, AND
extend `_collect_kit_files` to walk `HOST_SKILL_DIRS`.

### Blindspot review of this enhancement (mitigations baked in)

- **B-1. Checkbox staleness.** Mitigation: the resume rule says "re-run the
  phase's QA gate first; if it fails, uncheck and redo." The QA gate is the
  source of truth, not the checkbox. Documented in `session-continuity.md`.
- **B-2. Subagent race on checkboxes.** Mitigation: each subagent works a
  disjoint phase; the orchestrator checks cross-phase boxes. Documented.
- **B-3. Markdown checkbox format divergence.** Pick `- [ ]` (GitHub renders
  it; plain-text agents still parse `[ ]`). Apply consistently.
- **B-4. Plans without phases.** The gate enforces only on plans with >=2
  `## Phase` headings.
- **B-5. Audit trail.** The checkbox is a navigation aid, not an audit record.
  The phase's QA gate output (committed to `internal/owledge/exports/`)
  is the durable evidence. Documented.
- **B-6. Over-engineering risk.** Mandatory for multi-phase plans, optional
  for single-phase. The gate enforces only on >=2 phases.
