# Power-User Objections

Date: 2026-06-21  
Status: active pitch and product-memory source  
Audience: maintainers, agents, and reviewers preparing Owledge pitches or add-on plans.

## Purpose

This file preserves the strongest senior AI engineer objections raised during
product review. Treat these as durable pitch inputs and product guardrails.

## Core Objections

| Objection | Why it matters | Owledge response |
| --- | --- | --- |
| Why not just use a Markdown or Obsidian vault? | The core must prove it adds more than folders and notes. | Owledge adds typed memory, evidence links, review status, context packs, generated views, and promotion discipline while preserving Markdown. |
| Will this destroy my existing structure? | Power users already have project-specific `AGENTS.md`, skills, hooks, and vault taxonomies. | Default to principles-only, mapped KB, or add-on modules. Do not rewrite existing notes or wiki links. |
| Does this waste tokens? | Context pollution is a major AI engineering cost. | Scan metadata first, build scoped context packs, and prove savings with benchmark add-ons before making public claims. |
| Can it connect projects outside my central KB? | Real users keep repos, vaults, and experiments in separate folders. | Use project-local memory plus reviewed/sanitized cross-project promotion into a central hub or KB module. |
| Is it overengineered? | Too much process kills adoption. | Keep core minimal; ship proof, graph, hub, swarm, and benchmark capabilities as opt-in add-ons. |
| Can it coordinate agent swarms? | Codex, Claude Code, Hermes, and custom harnesses need shared state. | Use Owledge as coordination memory: goals, work lanes, evidence, handoffs, reviews, and curator promotion. Do not claim distributed locking in v1. |
| Can it show a decision tree? | Visual traceability is valuable for trust, onboarding, and AI Act-style audit trails. | Render read-only HTML and JSON traces from Markdown IDs, typed edges, evidence, reviews, decisions, and lessons. |

## Pitch Guardrails

- Do not claim universal token savings without benchmark evidence.
- Do not present Owledge as a hosted platform, vector database, or agent runtime.
- Do not imply automatic promotion from raw logs to canonical truth.
- Lead with the smallest integration path that solves the user's problem.
- Use add-ons to prove value without expanding core obligations.
