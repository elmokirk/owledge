# Handoff Report

## Use When

Use for PM to dev, dev to QA, agent to agent, orchestrator to worker, or client handoff requests.

## Sources

Prefer:

- `.owledge/handoffs/`
- `.owledge/sessions/`
- `.owledge/compiled/`
- `.owledge/evidence/`
- task cards or QA reports when present

## Sections

| Section | Purpose |
| --- | --- |
| Objective | What the next owner must achieve |
| Current state | What is done |
| Open decisions | What still needs choice |
| Blockers | What prevents progress |
| Write scope | Files or folders the next owner may touch |
| Validation | Commands or QA gates |
| Sources | Links and hashes |

## Visual Pattern

Use owner cards, checklist bands, risk callouts, and validation command blocks. Keep it short.
