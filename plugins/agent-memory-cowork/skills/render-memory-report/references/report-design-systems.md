# Report Design Systems

Use when a Cowork or Codex session asks for a report style, HTML report theme, visual selector, or persisted design-token decision.

## Rules

- Read `DESIGN.md` first.
- Use `REPORT_DESIGN_SELECTOR.html` only as a visual selection helper.
- Persist the chosen style by updating `selected_report_design` in `DESIGN.md`.
- Generated HTML must show source paths, hashes, report type, and selected design id.
- Markdown remains canonical; HTML is only a view.
